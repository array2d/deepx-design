# kvlang 指针语义设计分析：`&` / `*` / `null`

> C 语言有 `&` (取地址) / `*` (解引用) / `NULL`，因为它的内存是平坦字节数组，地址是整数。
> kvlang 的内存是 kvspace 树，地址是路径字符串。
> 虽然地址形式不同，但**指针的抽象语义完全适用**。

---

## 一、概念映射

| C 概念 | kvlang 等价物 | 语义 |
|--------|-------------|------|
| 内存地址 (整数) | kvspace 路径 (字符串) | 唯一标识一个存储位置 |
| `int x = 42` | `42 -> x` | 在 `frameRoot/x` 存储值 42 |
| `&x` → 取地址 | `&x` → `frameRoot/x` (路径字符串) | 获取变量 x 的路径 |
| `*p` → 解引用 | `*p` → `kv.Get(p)` | 读取路径 p 处的值 |
| `*p = v` → 间接写 | `v -> *p` 或 `*p <- v` | 写入路径 p 处 |
| `NULL` | `null` | 表示路径不存在 / 值为空 |
| `p != NULL` | `p != null` | 检查路径是否有效 |

---

## 二、语法

### 2.1 取地址 `&`

```
&x          → "frameRoot/x"          （获取变量 x 的路径）
&data[0]    → "frameRoot/data/0"     （路径下标构造）
&data[idx]  → "frameRoot/data/<idx>" （动态下标）
```

语义：返回变量的 kvspace 绝对路径字符串。不读值。

### 2.2 解引用 `*`

```
*p          → kv.Get(p)              （读取路径 p 处的值）
*p <- 42                              （写入路径 p 处）
v -> *p                               （同写入）
```

语义：`*` 后跟一个字符串表达式（路径），读取或写入该路径。

### 2.3 `null` 字面量

```
null        → Value{kind:"null"}     （表示不存在）
*p != null                           （检查路径是否有值）
```

对标 C 的 `NULL`、Go 的 `nil`、Rust 的 `None`。

## 三、Scanner / Parser 改动

### 3.1 Scanner

当前 `*` 是乘法算子 (Ident token)。需在 Pratt parser 中根据位置区分：
- 前缀位置 → 解引用 `*`
- 中缀位置 → 乘法 `*`（现有行为不变）

`&` 当前未在 scanner 中特殊处理，会被扫描为 Ident("&")。需添加单字符处理：
```go
case '&':
    tokens = append(tokens, Token{Kind: Ident, Value: "&", Pos: p})
```

### 3.2 Parser

`isUnaryPrefixOp` 扩展：
```go
func isUnaryPrefixOp(s string) bool { return s == "!" || s == "-" || s == "*" || s == "&" }
```

`parsePrimaryExpr` 中 `&` 的特殊处理：
- `&` 后跟标识符 → `Ref(ident)` → 返回路径字符串
- `&` 后跟 `ident[expr]` → `RefIndex(ident, expr)` → 构造 `path/idx`

`*` 的解析：
- `*` 后跟表达式 → `Deref(expr)` → 读取路径

C 语言中 `*` 既是解引用也是乘法，Pratt parser 天生支持这种区分（前缀 vs 中缀）。

## 四、内置算子

### 4.1 `ref` (opcode `&`)

```
输入：1 个读槽 — 变量名（字符串）
输出：1 个写槽 — 变量的绝对 kvspace 路径（字符串）
```

逻辑：`framePath + "/" + varName`

### 4.2 `ref.at` (opcode `&[]`)

```
输入：2 个读槽 — 前缀路径、整数索引
输出：1 个写槽 — "前缀/索引"（字符串）
```

逻辑：`prefix + "/" + strconv.Itoa(idx)`

等价于当前的 `kv.at` 的路径构造部分（不读值）。

### 4.3 `deref` (opcode `*`)

```
输入：1 个读槽 — 路径字符串
输出：1 个写槽 — 路径处的值；若不存在则为 null
```

逻辑：`kv.Get(path)`，不存在返回 `null`。

等价于：当前的 `kv.at` 去掉索引参数后的通用版本。

## 五、`kv.has` / `kv.at` → `&` / `*` 迁移

### 当前代码

```kvlang
for (v in data) {
    print(v)
}
```

lower 生成：
```
_for_cond:  _idx + 1 -> _idx;  kv.has(data, _idx) -> _cond;  br(_cond, body, exit)
_for_body:  kv.at(data, _idx) -> v;  ...
```

### 迁移后

```
_for_cond:  _idx + 1 -> _idx;  *(&data[_idx]) -> _val;  _val != null -> _cond;  br(_cond, body, exit)
_for_body:  *(&data[_idx]) -> v;  ...
```

或更简洁（编译器生成，用户不可见）：

```
_for_cond:  _idx + 1 -> _idx;  &data[_idx] -> _ptr;  *_ptr -> _val;  _val != null -> _cond;  br(_cond, body, exit)
_for_body:  &data[_idx] -> _ptr;  *_ptr -> v;  ...
```

`kv.has` 和 `kv.at` 可以删除，统一为 `&` + `*` + `null` 三个原语。

## 六、用户可见语法

### 6.1 直接指针操作

```kvlang
42 -> x                     # x = 42
&x -> p                     # p = 路径字符串 "/vthread/1/[0,0]/x"
*p -> v                     # v = 42 (读取 p 指向的值)
100 -> *p                   # 通过指针写入
```

### 6.2 动态路径构造

```kvlang
&data[0] -> p0              # p0 = "frameRoot/data/0"
&data[idx] -> ptr           # ptr = "frameRoot/data/<idx>" (动态下标)
*ptr -> val                 # 读取构造出的路径的值
```

### 6.3 存在性检查

```kvlang
*p != null -> exists         # C: if (p != NULL)
*p == null -> missing        # C: if (p == NULL)
```

### 6.4 遍历

```kvlang
def sum_all() -> (total:int) {
    0 -> total
    0 -> i
    loop {
        &data[i] -> ptr
        *ptr -> val
        if (val == null) { break }
        total + val -> total
        i + 1 -> i
    }
}
```

对应 C 代码：
```c
int sum_all() {
    int total = 0;
    for (int i = 0; ; i++) {
        int *p = &data[i];
        if (p == NULL) break;
        total += *p;
    }
    return total;
}
```

## 七、与 for 遍历器的关系

for 遍历器 `for (v in data) { ... }` 内部仍然使用 `&data[idx]` + `*ptr` 原语，
只是对外暴露为简洁的迭代语法。用户日常使用 for 即可，底层统一为指针原语。

```
用户写：for (v in data) { print(v) }
编译器展开：& + * + null 的 while 循环
```

## 八、实现路线

| 阶段 | 内容 |
|------|------|
| P1 | 添加 `&` / `*` / `null` 三原语（builtin op） |
| P2 | Scanner 支持 `&`；Pratt parser 支持前缀 `*` / `&` |
| P3 | `ref` / `deref` / `null` 运行时实现 |
| P4 | lower 迁移：`kv.has`/`kv.at` → `&`+`*`+`null` |
| P5 | 删除 `kv.has`/`kv.at`（内部统一） |

---

## 九、收益

1. **概念统一**：C 程序员直接理解，"地址就是路径字符串"
2. **精简原语**：`&` + `*` + `null` 替代 `kv.has` + `kv.at` + 未来的 `kv.len` 等
3. **动态能力**：`&data[idx]` 支持用户自行构造任意 kvspace 路径并读写
4. **对标 C**：`null` 是业界最广泛理解的"不存在"表示，优于 bool 标记位
