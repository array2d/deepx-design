# Tothink-006: 变量命名通货膨胀（读-算-用的三步舞）

**严重程度**：🟢 低 — 认知负担不高但代码冗长，与 agent-001 联动。

---

## 现象

两条语句的简单操作（可直接读→算→写回同名变量）：
```kvlang
total + i -> total         # 读 total,i，算加法，写回 total — 无需中间变量
```

三条语句的条件判断：
```kvlang
x + y -> s                 # 1. 读 x,y，算 x+y，写 s
s == target -> hit         # 2. 读 s，算比较，写 hit
if (hit) { ... }           # 3. 读 hit，分支
```

对比 Python：
```python
if x + y == target:        # 一行
    ...
```

## 影响

agent 需要多产生 1-2 个临时变量名（`s`, `hit`, `divisible`, `found`, `cond`...）。在复杂算法中变量数量膨胀，agent 需要额外脑力管理命名空间。

不是 bug — 是读写码"每条指令的操作数为叶节点"约束的设计代价。

## 缓解

与 agent-001（`if (expr)` 支持）联动。当条件表达式被 lower 自动展开时，中间变量由编译器生成（`_t0`, `_cond_0`），agent 无需手动命名。

## 建议

优先级低。优先实现 agent-001（条件表达式自动展开），本问题自然缓解。
