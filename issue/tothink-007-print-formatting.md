# Tothink-007: print 输出格式粗糙，无字符串插值

**严重程度**：🟢 低 — 不影响正确性，影响输出质量和 agent 友好度。

---

## 现象

```kvlang
print("[", i, ",", j, "]")   # 输出: [ 0 , 1 ]
```

逗号分隔的参数自动以空格连接，导致：
- `[ 0 , 1 ]` 多了不必要的空格
- 无法控制字段宽度、对齐、精度
- 无法嵌入表达式（如 `print("sum =", a + b)` — `a + b` 无具名写槽，必须分两步）

## agent 直觉

```python
print(f"[{i}, {j}]")         # f-string
print("[%d, %d]" % (i, j))   # printf-style
```

## 影响

1. LeetCode 验证通常匹配宽松（可以接受空格差异），但正式场景下输出格式不可控
2. 调试时 `print("x =", ./x)` 形式够用，但格式化输出无法实现
3. `print` 的参数必须是叶节点，`print("sum =", a + b)` 不合法

## 建议

长期优化项。可能的方案：
- A：`print` 支持格式化字符串 `print("[%d, %d]", i, j)`
- B：支持 `cat` 字符串拼接 `cat("[", i, ",", j, "]") -> s; print(s)`
- C：支持模板字符串 `` `` `[${i}, ${j}]` `` ``
